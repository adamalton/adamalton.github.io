Safari CSP Bug Demo

This page demonstrates a bug in Safari (existing in version 13.1.1 on macOS and whatever version is running on iOS 13.5) where the browser's native <audio> element is adversely affected by a page's Content Security Policy.

Steps to reproduce

  1. Create an HTML page with an element <audio controls><audio>.
  2. Serve that page with a Content-Security-Policy where the img-src (or the implied img-src via default-src) does not permit data.
  3. Load the page in Safari.

Observed Behaviour

Safari displays the audio element, but the control buttons are missing.

Expected Behaviour

The Content Security Policy should not affect the behaviour of native browser elements.
